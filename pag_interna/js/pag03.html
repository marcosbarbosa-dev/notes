OPERADOR ARITMÉTICOS

OPERADOR ARITMÉTICOS

Usados para cálculos:
+ // Adição
- // Subtração
* // Multiplicação
/ // Divisão Real
% // Divisão Inteira
** // Potência (surgiu com ECMAScript)

São chamados de operadores binários, pois precisam de dois operandos.

EXEMPLO:
5 + 2 // 7
5 - 2 // 3
5 * 2 // 10
5 / 2 // 2.5
5 % 2 // 1 (É o resto da divisão de 5 / 2)
5 ** 2 // 25 (5 ao quadrado)

ORDEM DE PRECEDÊNCIA

ORDEM DE PRECEDÊNCIA

Caso em uma conta, eu fizer 5 + 3 / 2, o resultado é igual a 6.5, já que a divisão vem antes da adição.

Mas se eu quiser realmente realizar primeiro a adição e depois a divisão?

Coloca-se a soma entre parenteses.
Exemplo: (5 + 3) / 2 = 4

OPERADOR DE ATRIBUIÇÃO SIMPLES

OPERADOR SIMPLES

var a = 5 + 3 (8) // a varível 'a' recebe '=' o valor de 5 + 3.
var b = a % 5 (8)
var c = 5 * b ** 2 (45)
var d = 10 - a / 2 (6)
var e = 6 * 2 / d (2) // como ambos tem a mesma precedência, o que vem primeiro é calculado.
var f = b % e + 4 / e (3)
Os operadores de atribuição simples são os de variáveis

AUTO ATRIBUIÇÃO

var n = 3 // atribuição simples
n = n + 4 // auto atribuilção

Agora o 'n' deixará de valor 3 e comecará a valer 7.

SIMPLIFICADO

n = n + 4 // auto atribuição
n += 4 // simplificado

Só serve se a mesma variável receber ela mesma mais o outro valor.
N = N + ...

OPERADOR DE INCREMENTO

OPERADOR DE INCREMENTO

Como no exemplo anterior de simplificação, se eu tenho:

var x = 5 // atribuição simples
x = x + 1 // auto atribuição
x += 1 // auto atribuição simplificado
x++ // operador de incremento

Todos valores serão os mesmos

EXEMPLO:

var x = 5
x = x + 1 // resultado: 6
x++ // resultado: 6 incrementou um número)

x = x - 1 // resultado: 4
x-- // resultado: 4 (descrementou um número)

OPERADOR RELACIONAIS

OPERADOR RELACIONAIS

> // maior que
< // menor que
>= // maior ou igual
<= // menor ou igual
== // igual
!= // diferente

5 > 2 // true
7 < 4 // false
8 >= 8 // true
9 <= 7 // false
5 == 5 // true
4 != 4 // true

O resultado dessas expressões serão sempre valores booleanos, ou seja, verdadeiro ou falso.

Booleanos: true ou false

OPERADOR DE IDENTIDADE

OPERADOR DE IDENTIDADE

5 == 5 // true

5 == '5' // true, pois o sinal de igualdade não testa o tipo, apenas analisa se o 5 tem o mesmo valor que o outro. Mesmo sendo de tipos diferentes, possuem a mesma grandeza.

5 === '5' // false (operador de identidade ou de igualdade restrita), pois estamos testando se os dois 5 da direita são identidos (mesmo valor e mesmo tipo). Nesse caso, tem o mesmo valor mas não o mesmo tipo.

5 === 5 // true

OPERADOR LÓGICO

OPERADOR LÓGICO

! // negação / Exemplo: Quero qualquer caneta, menos azul.

& // conjunção / Exemplo: Quero uma caneta azul e preta, não serva só uma.

|| // disjunção / Exemplo: Quero uma caneta azul e uma vermelha, mas se eu receber só uma delas, tá ótimo. Mas se não me entregar nenhuma ou de uma cor que eu não queira, não serve.

OPERADOR DE NEGAÇÃO

OPERADOR DE NEGAÇÃO

! O 'não' // É tratado como operador unário, ele só possui um operando. Depois da exclamação, ou coloco o true ou o false. Ou coloco uma expressão que vai dar true ou false.

OPERADOR DE CONJUNÇÃO

OPERADOR DE CONJUNÇÃO

&& // É um operador binário, nele tenho dois valores lógicos, um de cada lado.

EXEMPLO:
true && true -> true true && false -> false false && true -> false false && false -> false

Só será verdadeiro se as duas forem verdadeiras.

OPERADOR DE DISJUNÇÃO

OPERADOR DE DISJUNÇÃO

| // Operador binário, que dependendo da posição deles e do valor lógico, eu terei um valor lógico de resultados.

EXEMPLO:
true || true -> true true || false -> true false || true -> true false || false -> false

Basta que apenas um seja verdadeiro para que a resposta seja verdadeira. Apenas se os forem falsos, aí sim o resultado será falso.

ORDEM DE PRECEDÊNCIA
!
&&
||

EXEMPLOS DE EXPRESSÕES
idade >= 15 && idade <= 17 // a idade 'está entre' 15 'e' 17?
estado == 'RJ' || estado == 'SP' // o estado 'é' RJ 'ou' SP?
salário > 1500 && sexo != 'M' // o salário 'é maior que' 1500 e o diferente de masculino?
